# Topic Filters

Apply these filters before scoring.

## Must-Have Criteria

Accept a candidate only if it has at least three:

- Clear AI connection.
- Recent or newly resurfaced reason to talk about it.
- Strong user implication for AI game development, AI monetization, or AI tool practice.
- Concrete demo, product, repo, benchmark, pricing, case, or data.
- Visual material suitable for short video: product page, demo clip, UI, chart, GitHub, before/after, workflow.
- A non-obvious viewpoint that helps long-term creator authority.

## AI Game Development Criteria

Accept game development topics when AI materially changes:

- asset generation: 3D, textures, animation, environment, characters.
- production workflow: prototype, level design, world generation, testing, localization.
- gameplay systems: AI NPCs, agents, dynamic worlds, procedural quests.
- solo/small-team feasibility.
- engine integration with Unity, Unreal, Roblox, Omniverse, or similar tools.

Reject ordinary game industry news unless it has a real AI production, AI gameplay, or AI business-model angle.

## Long-Term Persona Filter

Prefer topics that let the creator sound like:

- practical, not sensational.
- early but skeptical.
- builder-oriented.
- focused on how AI changes workflows, income, and independent creation.
- willing to explain limits.

Penalize:

- "AI replaces everyone tomorrow" panic.
- unsupported income claims.
- pure model leaderboard content without application.
- generic press-release summaries.
- topics that only create heat but no useful takeaway.

## Duplicate Handling

If multiple sources describe the same event, merge them into one candidate. Keep the strongest source links and use the extra sources as evidence.

## Candidate Rewrite Rule

Always rewrite raw news into a creator-facing short-video angle.

Bad: `Company X released model Y`

Good: `游戏开发的原型阶段正在被 AI 重写：一个人做 3D 世界的门槛又降了一截`
